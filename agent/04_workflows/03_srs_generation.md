# Workflow: SRS Generation

## Purpose

Convert vague ideas into rigid, testable requirements.

## Inputs

- Project context from Intake.

## Procedure

1. **Adopt Profile**: `profile_analyst_srs.md`.
2. **Drafting**: Use `07_templates/srs/` to fill `functional_reqs.md` and `non_functional_reqs.md`.
3. **User Stories**: Map requirements to User Stories.
4. **Validation**: Check for ambiguity (e.g., words like "Fast", "Easy", "Scalable" without metrics).
5. **User Review**: Present the draft for approval.
6. **Gate**: Verify `05_gates/global/gate_global_docs.md`.

## STOP Condition

- User says "SRS Approved".
