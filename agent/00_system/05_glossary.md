# Glossary

## Purpose

Provides a standard vocabulary for the Agent OS to ensure clarity across files.

## Terms

- **Agent**: The AI model operating this system.
- **Backlog**: The list of prioritized tasks and features yet to be implemented.
- **Baseline**: The "known good" state of a repository before a change.
- **Checkpoint**: A minor state save before a risky operation.
- **Gate**: A mandatory validation point (quality, security, or logic).
- **Fingerprint**: Identifying markers of a technology stack (e.g., `package.json` for Node).
- **Implementation Loop**: The iterative process of code -> test -> fix.
- **Plan Output Contract (POC)**: The agreement on where files are saved.
- **Profile**: A set of persona-based instructions for the AI.
- **SRS**: Software Requirements Specification.
- **Stack**: The combination of OS, Language, Framework, and Database.
- **State**: The current lifecycle position and health of the project.
- **Workflow**: A sequence of tasks to achieve a specific SDLC goal.

## Acronyms

- **ERD**: Entity Relationship Diagram.
- **DFD**: Data Flow Diagram.
- **SDLC**: Software Development Lifecycle.
- **PR**: Pull Request.
- **CI/CD**: Continuous Integration / Continuous Deployment.
