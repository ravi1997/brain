# Templates Index

## Purpose

A central registry for all document and code templates used by the Agent.

## Central Templates (`07_templates/`)

### Specification & Design

- `srs/functional_reqs.md`: Template for features and user stories.
- `srs/non_functional_reqs.md`: Template for performance, security, and scale.
- `diagrams/erd_template.mermaid`: For database design.
- `diagrams/flowchart_template.mermaid`: For logic and process flows.

### Implementation

- `feature/feature_kickoff.md`: Template for starting a new developer task.
- `feature/implementation_summary.md`: Template for reporting finished work.

### Quality & Governance

- `pr/pull_request_template.md`: Format for peer reviews.
- `release/release_notes_template.md`: Format for final project delivery.
- `security/threat_model_template.md`: Format for analyzing risks.

## State Templates (`09_state/`)

- `PROJECT_STATE.md`: The master lifecycle tracker.
- `BACKLOG_STATE.md`: The task list tracker.

## Usage Rule

The Agent must ALWAYS copy the raw content of these templates before filling them to ensure no headers or structural elements are missed.
