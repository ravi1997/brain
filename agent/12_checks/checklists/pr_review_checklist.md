# PR Review Checklist

## Correctness

- [ ] Does the code satisfy the SRS requirement?
- [ ] Are all edge cases handled?
- [ ] Is the logic sound and efficient?

## Quality

- [ ] Are naming conventions followed?
- [ ] Is the code DRY?
- [ ] Are variable scopes minimized?

## Testing

- [ ] Do all tests pass?
- [ ] Is there sufficient coverage?

## Security

- [ ] No hardcoded secrets?
- [ ] Input validation is present?
