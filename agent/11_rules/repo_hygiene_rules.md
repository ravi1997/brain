# Repo Hygiene Rules

## 1. Folder Structure

- Source code in `src/` or `app/`.
- Assets in `assets/` or `public/`.
- Configs in root.
- Plans in `plans/`.
- Agent state in `agent/`.

## 2. Git Standards

- **Commit Messages**: Use Conventional Commits (`feat:`, `fix:`, `docs:`, `test:`).
- **Branching**: Use feature branches (`feature/login`, `bugfix/api-error`).

## 3. File Metadata

- Every new file must have a 2-line header explaining its purpose.
