# Template: .dockerignore

## Purpose

This template provides a comprehensive `.dockerignore` file to exclude unnecessary files from Docker build context, reducing image size and build time.

## Notes

- Always exclude `.env` files to prevent secrets from being baked into images
- Exclude test files and coverage reports to reduce image size
- Exclude documentation that's not needed at runtime
- Keep lock files (package-lock.json, yarn.lock, requirements.txt) as they're needed for reproducible builds
- Exclude IDE and editor configuration files
- Exclude CI/CD configuration files

## Validation

To see what files are being sent to Docker build context:

```bash
docker build --no-cache --progress=plain . 2>&1 | grep "COPY"
```

Or check the build context size:

```bash
docker build --no-cache . 2>&1 | grep "Sending build context"
```
